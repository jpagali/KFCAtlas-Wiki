# Release 2026.03.14

Released on **March 14, 2026**.

This release focused on cleanup and stability. The main goal was to make localization feel reliable in practice, especially when moving between English and Japanese while browsing real documentation pages.

## Highlights

- **Locale switching now follows the current page**
  The language switcher now uses Docusaurus' page-aware alternate URL logic, so switching between EN and JP keeps you on the equivalent route instead of relying on brittle path rewriting.

- **Japanese documentation coverage was filled in where it was visibly broken**
  Missing Japanese survey pages and related overview references were added so key front-end documentation paths resolve correctly.

- **Local testing is simpler and less confusing**
  The temporary dual-server dev workaround was removed. The docs now point people toward a cleaner workflow: use a normal locale-specific dev server for editing and a single-port built preview for cross-locale testing.

- **Changelog navigation was decluttered**
  Release notes remain available in the footer without taking space in the left documentation sidebar.

## What Changed

### Localization

- Replaced manual locale URL rewriting with Docusaurus alternate-page resolution in the navbar locale switcher
- Improved Japanese footer translation coverage for newly added footer items
- Added Japanese customer engagement survey pages, including Qualtrics
- Updated the Japanese front-end overview so newer sections are represented consistently

### Site Experience

- Removed release notes from the front-end left navigation
- Kept changelog access in the footer only
- Rebuilt the static output so the latest locale behavior is reflected in the current site assets

### Developer Experience

- Removed the temporary multi-server locale launcher
- Simplified local scripts back to a cleaner default setup
- Clarified in the README that one-port built preview is the right way to test EN/JP switching

## In Plain English

This was the cleanup release. The site now switches languages more predictably, the Japanese paths that were obviously missing have been filled in, and the local workflow is easier to understand without carrying around experimental setup code.
